자주 묻는 질문

궁금하신 점을 빠르게 확인하세요.

중요제품/솔루션HPC(고성능 컴퓨팅)란 무엇이며, 일반 서버와 무엇이 다른가요?
+
HPC(High Performance Computing)는 수천 개 이상의 CPU·GPU 코어를 네트워크로 연결해 대규모 연산을 병렬로 처리하는 컴퓨팅 환경입니다.

일반 서버와의 주요 차이점:
• 연산 규모: 일반 서버는 단일 노드 처리, HPC는 수십~수천 노드를 묶어 하나의 거대한 연산 자원처럼 활용
• 인터커넥트: HPC는 InfiniBand, HDR/NDR 등 초저지연·고대역폭 네트워크 필수 (일반 서버는 이더넷 사용)
• 스토리지: 수십 GB/s 이상의 병렬 파일시스템(Lustre, VAST Data 등) 필요
• 활용 분야: 기상·해양 수치모델, 유체역학 시뮬레이션, AI 학습, 유전체 분석, 핵융합 연구 등

VWorks Technologies는 국방·연구기관에 HPE Cray 기반 HPC 클러스터를 구축한 경험을 보유하고 있습니다.
중요도입/견적VAST Data 스토리지 도입 비용은 얼마인가요?
+
VAST Data는 고객사 규모와 요구사항에 따라 커스터마이징 견적을 드리고 있습니다. 용량과 환경에 따라 상이하오니 모든 문의하기 양식을 통해 무료 견적을 요청해 주세요.
중요제품/솔루션AI 인프라와 일반 HPC 인프라의 차이점은 무엇인가요?
+
AI 인프라와 전통적 HPC 인프라는 목적과 구성 요소에서 차이가 있습니다.

구분 | HPC 인프라 | AI 인프라
연산 중심 | CPU 중심 (수치 연산) | GPU 중심 (행렬 연산·딥러닝)
네트워크 | InfiniBand (MPI 통신) | InfiniBand + NVLink (GPU간 직접 통신)
스토리지 | 병렬 파일시스템 (Lustre) | 고속 NVMe + 오브젝트 스토리지 (VAST Data)
워크로드 | 시뮬레이션, CFD, FEA | 모델 학습, 추론, RAG, 에이전트 AI

최근에는 AI+HPC 통합 환경(AI for Science)이 주류가 되어, 기상예측·단백질 구조 분석 등에 GPU 클러스터가 활용됩니다.

VWorks는 HPE Cray EX(HPC)와 HPE ProLiant DL GPU 서버(AI)를 모두 공급합니다.
중요제품/솔루션GPU 서버 도입 시 NVIDIA A100, H100, H200의 차이는 무엇인가요?
+
NVIDIA GPU 세대별 비교 (AI 학습·추론 기준):

• A100 (Ampere, 2020)
- 메모리: 80GB HBM2e
- 학습 성능: 312 TFLOPS (BF16)
- 인터커넥트: NVLink 3세대, 600GB/s
- 적합: LLM 파인튜닝, 중소규모 AI 학습

• H100 (Hopper, 2022)
- 메모리: 80GB HBM3
- 학습 성능: 989 TFLOPS (BF16), A100 대비 3배↑
- 인터커넥트: NVLink 4세대, 900GB/s
- 적합: 대규모 LLM 사전학습, 고속 추론

• H200 (Hopper+, 2024)
- 메모리: 141GB HBM3e (H100 대비 76% 증가)
- 추론 성능: H100 대비 최대 2배↑
- 적합: 초대형 모델(70B+) 추론, 멀티모달 AI

예산과 워크로드에 따라 최적 GPU를 선택해야 하며, VWorks는 HPE ProLiant DL380a(H200 NVL×8) 등 최신 GPU 서버를 공급합니다.
제품/솔루션HPC 클러스터 구축 시 InfiniBand와 이더넷 중 무엇을 선택해야 하나요?
+
HPC·AI 클러스터의 인터커넥트 선택 기준입니다.

InfiniBand (권장: HPC·AI 학습)
• 지연 시간: 600ns 이하 (이더넷 대비 10배 낮음)
• 대역폭: NDR 400Gb/s, HDR 200Gb/s
• MPI 통신 최적화 → 병렬 시뮬레이션 성능 극대화
• RDMA 지원 → GPU 메모리 직접 접근 (GPUDirect)
• 단점: 고비용, 전용 스위치 필요

이더넷 (권장: 추론 서버·비용 민감 환경)
• 100GbE~400GbE RoCE v2 RDMA 지원
• 범용 스위치 활용 가능 → 비용 절감
• 관리 용이성 높음
• 단점: InfiniBand 대비 지연 시간 높음

권장 기준:
- 기상/해양 수치모델, LLM 학습 → InfiniBand NDR/HDR
- AI 추론 서버, 소규모 클러스터 → RoCE 이더넷

구체적인 요구사항에 맞는 네트워크 설계를 VWorks가 지원합니다.
중요제품/솔루션VAST Data 스토리지가 일반 NAS·SAN과 다른 점은 무엇인가요?
+
VAST Data는 기존 스토리지 계층 구조(Hot/Warm/Cold)를 완전히 제거한 차세대 통합 데이터 플랫폼입니다.

기존 NAS·SAN과의 핵심 차이:

• 성능 vs 용량 트레이드오프 없음
기존: 빠른 SSD(소용량) vs 느린 HDD(대용량) 티어 관리 필요
VAST: 전체 용량을 NVMe 속도로 제공 (PB급도 가능)

• 단일 네임스페이스
수백 PB의 데이터를 단일 파일시스템으로 관리
AI 학습 데이터셋, HPC 체크포인트 등 통합 접근

• 다중 프로토콜 동시 지원
NFS, S3, SMB, POSIX를 동시에 지원
AI 프레임워크(PyTorch, TensorFlow)와 HPC(MPI) 동시 활용

• AI OS 기능 내장
벡터 DB, 내장 데이터베이스, 워크플로우 트리거 등
데이터 파이프라인을 스토리지 안에서 처리

국방·연구기관의 대용량 기상 데이터, HPC 체크포인트, AI 학습 데이터셋에 최적화된 솔루션입니다.
중요기술지원기상·해양 수치모델링에 필요한 HPC 사양은 어떻게 됩니까?
+
기상·해양 수치모델(WRF, ROMS, MOM6, NEMO 등) 운영을 위한 권장 사양입니다.

컴퓨팅 노드:
• CPU: Intel Xeon Scalable (Ice Lake/Sapphire Rapids) 또는 AMD EPYC
• 코어 수: 노드당 32~128코어 (MPI 병렬화 최적)
• 메모리: 노드당 256GB~512GB (모델 해상도에 따라)
• 최소 노드 수: 해상도 1km 기상모델 기준 64노드 이상 권장

인터커넥트:
• InfiniBand HDR 200Gb/s 이상 (MPI 통신 최적화)
• Fat-tree 토폴로지 권장

스토리지:
• 병렬 파일시스템: VAST Data 또는 Lustre
• 처리량: 초당 100GB/s 이상 (고해상도 모델 I/O)
• 용량: 장기 데이터 보관 고려 PB급 설계

소프트웨어 스택:
• OS: Rocky Linux / RHEL
• MPI: OpenMPI / MPICH
• 스케줄러: SLURM
• 환경관리: Lmod + Spack

VWorks는 해군·기상청 등 국내 기상해양 기관에 HPC 구축 레퍼런스를 보유하고 있습니다.
제품/솔루션HPE Cray EX 슈퍼컴퓨터와 일반 클러스터의 차이는 무엇인가요?
+
HPE Cray EX는 페타플롭(PF)급 국가 슈퍼컴퓨터를 위한 고집적 컴퓨팅 플랫폼입니다.

일반 HPC 클러스터와의 차이:

항목 | 일반 클러스터 | HPE Cray EX
냉각 방식 | 공랭(Air Cooling) | 직접 수냉(Direct Liquid Cooling)
전력 밀도 | 20~30kW/랙 | 100kW+/랙 (4~5배)
인터커넥트 | InfiniBand / 이더넷 | Slingshot (HPE 독자 패브릭)
관리 소프트웨어 | 범용 HPC SW | CSM (Cray System Management)
GPU 지원 | 선택적 | NVIDIA H100/H200, AMD MI300 통합
규모 | 수십~수백 노드 | 수천~수만 노드

Slingshot 인터커넥트 특징:
• 200Gb/s 포트 속도
• 적응형 라우팅으로 네트워크 혼잡 자동 회피
• AI 학습과 HPC 혼용 환경 최적화

국가 기상청, 핵융합연구원, 방위산업체 등 대규모 연산 수요 기관에 적합합니다.
중요기술지원LLM(대형 언어모델) 학습을 위한 최소 GPU 수량은 얼마나 필요한가요?
+
LLM 학습에 필요한 GPU 수량은 모델 크기에 따라 크게 달라집니다.

모델 규모별 최소 GPU 권장 사양 (H100 80GB 기준):

• 7B 파라미터 모델
- 풀 파인튜닝: 4× H100
- LoRA 파인튜닝: 1× H100
- 사전학습: 8× H100 이상

• 13B 파라미터 모델
- 풀 파인튜닝: 8× H100
- 사전학습: 16× H100 이상

• 70B 파라미터 모델
- 풀 파인튜닝: 32× H100
- 사전학습: 64× H100 이상

• 405B+ 파라미터 모델 (GPT-4급)
- 사전학습: 512× H100 이상

학습 시간 단축 팁:
- DeepSpeed ZeRO-3, Megatron-LM 등 분산학습 프레임워크 활용
- 혼합 정밀도 학습(BF16/FP8)으로 메모리 효율 향상
- 고속 스토리지(VAST Data)로 데이터 로딩 병목 제거

VWorks는 GPU 서버 공급부터 분산학습 환경 구성까지 통합 지원합니다.
중요도입/견적HPC·AI 시스템 도입 시 전력·냉각 설비도 함께 준비해야 하나요?
+
HPC·AI 시스템은 일반 IT 장비보다 전력 밀도가 높아 반드시 사전 인프라 검토가 필요합니다.

전력 요구사항:
• GPU 서버(8× H100): 랙당 10~15kW
• HPE Cray EX 블레이드: 랙당 30~60kW
• 전체 클러스터: 수십~수백 kW (변전설비 용량 확인 필수)

냉각 방식별 특징:
• 공랭(Air Cooling)
- 일반 데이터센터 활용 가능
- 랙당 최대 20~25kW 처리
- 고밀도 GPU 클러스터에는 부적합

• 직접 수냉(Direct Liquid Cooling, DLC)
- 랙당 50~100kW+ 처리 가능
- HPE Cray EX, NVIDIA DGX H100 등 고밀도 시스템 필수
- 냉각 배관 설치 공사 필요

• 침지 냉각(Immersion Cooling)
- 최고 효율 PUE 1.03 이하
- 대규모 데이터센터 신축 시 고려

도입 전 체크리스트:
✅ 기존 전력 인입 용량 확인
✅ UPS 용량 검토
✅ 냉각 시스템 설계 (공랭/수냉)
✅ 바닥 하중 검토

VWorks는 하드웨어 공급과 함께 전력·냉각 요구사항 컨설팅도 제공합니다.
중요도입/견적HPC 클러스터 구축 견적은 어떻게 산정되나요?
+
HPC 클러스터 구축 비용은 아래 항목들로 구성됩니다.

주요 비용 구성:

1. 컴퓨팅 노드
- CPU 서버: 노드당 500만~2,000만원
- GPU 서버(H100 8장): 노드당 4억~6억원
- HPE Cray EX 블레이드: 별도 견적

2. 인터커넥트 (네트워크)
- InfiniBand HDR/NDR 스위치: 규모에 따라 수천만~수억원
- 케이블링 포함

3. 스토리지
- VAST Data (1PB 기준): 수억~수십억원
- Lustre 병렬 파일시스템 구성 포함

4. 소프트웨어 라이선스
- SLURM, PBS 스케줄러
- 컴파일러, MPI, 수학 라이브러리

5. 설치·구축·유지보수
- 현장 설치, 네트워크 구성, OS 설치
- 연간 유지보수 계약

정확한 견적을 위해 필요한 정보:
• 목표 연산 성능 (TFLOPS/PFLOPS)
• 활용 예정 소프트웨어 및 워크로드
• 설치 장소 전력·냉각 현황
• 예산 범위

문의하기 페이지를 통해 요구사항을 전달해주시면 맞춤 견적을 제공해드립니다.

중요제품/솔루션HPC에서 주로 사용하는 병렬 파일시스템의 종류와 각각의 특징은 무엇인가요?
+
HPC 환경에서는 수백~수천 개의 노드가 동시에 스토리지에 접근하는 워크로드 특성상, 일반 NAS와는 전혀 다른 병렬 파일시스템이 필요합니다.

【 Lustre 】
• 오픈소스 기반의 HPC 병렬 파일시스템 (GPL 라이선스)
• 전 세계 TOP500 슈퍼컴퓨터의 70% 이상이 채택
• 초당 수백 GB/s ~ TB/s급 대역폭 구현 가능
• MDS(메타데이터 서버) + OSS(스토리지 서버) 분리 아키텍처
• 장점: 검증된 안정성, 방대한 커뮤니티, 무료
• 단점: 설치·운영 복잡, 전문 인력 필요, 상용 지원 별도 비용

【 GPFS (IBM Spectrum Scale) 】
• IBM의 엔터프라이즈 병렬 파일시스템
• 단일 네임스페이스로 수십 PB 관리
• Active File Management(HSM) 지원
• 장점: 엔터프라이즈 안정성, 다양한 프로토콜 지원, IBM 공식 지원
• 단점: 상용 라이선스 비용, IBM 하드웨어 종속성

【 WEKA 】
• NVMe 기반 차세대 병렬 파일시스템
• POSIX 완전 호환, 소프트웨어 정의 스토리지
• GPU Direct Storage 지원으로 AI 워크로드 최적화
• 장점: 초저지연(수십 마이크로초), 설치·관리 용이, 클라우드 하이브리드
• 단점: 상용 라이선스 비용, 상대적으로 신생 솔루션

【 VAST Data 】
• 차세대 올플래시 통합 데이터 플랫폼
• 스토리지 계층 구조(Hot/Warm/Cold) 완전 제거
• NFS, S3, SMB, POSIX 동시 지원
• 내장 벡터 DB, 데이터베이스, AI OS 기능
• 장점: 관리 단순, PB급에서도 NVMe 속도, AI+HPC 통합 최적
• 단점: 상용 솔루션으로 도입 비용 존재

✅ VAST Data가 Lustre 대비 우수한 점
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• 운영 단순성: Lustre는 MDS/OSS 분리 아키텍처로 운영 복잡, VAST는 단일 플랫폼으로 관리 용이
• 스토리지 티어링 불필요: Hot/Warm/Cold 계층 관리 없이 전체 용량을 NVMe 속도로 제공
• 다중 프로토콜: NFS·S3·SMB·POSIX를 동시에 지원해 HPC와 AI 워크로드를 단일 플랫폼에서 처리
• AI OS 기능: 벡터 DB, 내장 데이터베이스, 워크플로우 엔진 등 데이터 파이프라인 내재화
• 소프트웨어 스택 단순화: Lustre 클라이언트 설치·패치 관리 불필요
